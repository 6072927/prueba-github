Git

Analogia:

Git es como como un diario. Cada vez que haces un “commit” sería como escribir una

nueva entrada o tomar una foto que queda registrada como un momento en el tiempo.

Las “branches” serían como diferentes caminos o tareas en la vida, y luego puedes “mergear”

o juntar caminos cuando decidas hacer que esos eventos se conecten. El “repositorio”

sería tu gran proyecto de vida, almacenando todas esas versiones y decisiones para

regresar a ellas cuando quieras.
